# AirBrB 后台管理前端（Admin Frontend）

> 基于 React 的中台 / 后台管理前端项目，模拟真实企业级管理平台，聚焦 CRUD 业务流、组件复用与前后端解耦。

---

## 📌 项目简介

本项目是一个使用 **React** 构建的 **单页后台管理系统（SPA）**，  
用于模拟真实企业级 **中台 / 后台管理平台** 的前端实现。

系统覆盖常见管理端能力，包括：

- 用户认证
- 内容（房源）管理
- 运营配置
- 多条件搜索与筛选
- 稳定、统一的交互反馈机制

项目从**前端工程实践**角度出发进行设计与实现，而非简单的页面展示或课程示例。

---

## ✨ 核心功能

### 🧩 后台管理能力

- 用户登录 / 注册与鉴权
- 房源全生命周期管理  
  - 创建 / 编辑  
  - 发布 / 下架
- 可用日期配置
- 评论与评分展示
- 多条件搜索与排序：
  - 价格
  - 房间数
  - 评分
  - 日期区间

---

### 🛠️ 前端工程设计

- 单页应用（SPA）架构
- 基于 **React Router** 的清晰路由设计  
  - 列表页  
  - 详情页  
  - 编辑页  
  - 新建页
- 统一处理：
  - 加载态（Loading）
  - 空态（Empty）
  - 错误态（Error）
- 全局 Snackbar 消息反馈，避免页面级重复逻辑

---

## 🧱 技术栈

- **前端框架**：React（Hooks）
- **语言**：JavaScript（ES6）
- **路由**：React Router
- **UI 组件库**：Material UI（MUI）
- **样式**：CSS
- **接口通信**：HTTP + JSON（后端 API）
- **版本管理**：Git

---

## 🧩 组件设计与复用

项目采用 **组件驱动的开发方式**，强调复用性与一致性：

- **ListingForm**  
  用于房源创建与编辑的通用表单组件，在多个页面中复用。

- **FormTextField**  
  对输入框进行统一封装，抽象校验、标签与可访问性属性。

- **全局 Snackbar**  
  集中处理成功 / 失败 / 警告等反馈信息，避免各页面重复实现提示逻辑。

---

## 🗂️ 路由与页面结构

使用 **React Router** 管理页面路由，页面职责划分清晰：

- 管理列表页（Overview）
- 详情展示页
- 编辑页
- 新建页

该结构贴近真实企业后台系统，便于后续功能扩展与维护。

---

## 🔄 前后端协作方式

- 通过 HTTP 请求与后端 API 进行数据交互
- 统一处理鉴权 Header 与错误返回
- 避免在 UI 组件中硬编码接口结构
- 保证数据一致性与系统稳定性

---

## ♿ 用户体验与可访问性（A11Y）

项目中引入基础 **UX 与 A11Y** 设计原则：

- 表单语义化结构
- 支持键盘操作

- 统一的交互规范
